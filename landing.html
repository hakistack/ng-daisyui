WHY NG-DAISYUI

Built the Right Way

Angular 21 best practices baked in. No shortcuts, no legacy patterns.

Strict TypeScript

Full strict mode with strict templates and injection parameters. No any types.

Signals-First

Built on Angular signals: signal(), computed(), input(), output().

OnPush Everywhere

Every component uses ChangeDetectionStrategy.OnPush for peak performance.

Standalone Components

No NgModules. Every component is standalone and tree-shakeable by default.

Accessible

WCAG AA compliant with full keyboard navigation, ARIA attributes, and focus management.

DaisyUI v4 + v5 Themes

Full theme support with provideHkTheme(). Swap themes at the provider level.

Reactive Forms

All form logic is reactive. No template-driven forms. Clean, predictable state management.

CDK DataSource

Table integrates with Angular CDK DataSource for server-side data, streaming, and custom loading strategies.

Auto-Save

Form state persists to localStorage or your API with debounced saves via FormStateService.

BUILDER API

Declarative & Type-Safe

Configure components with builder functions. Full IntelliSense, zero guesswork.

Form Builder 

const form = createForm({
  fields: [
    field.text('name', 'Full Name', {
      required: true,
      minLength: 2,
      placeholder: 'John Doe'
    }),
    field.email('email', 'Email'),
    field.select('role', 'Role', {
      choices: ['Admin', 'Editor', 'Viewer']
    }),
    field.toggle('active', 'Active'),
  ],
  layout: 'grid',
  gridColumns: 2,
  onSubmit: (data) => console.log(data),
});
// Template: <hk-dynamic-form [config]="form.config()" />

Table Builder 

const table = createTable<User>({
  visible: ['id', 'name', 'email', 'role', 'salary'],
  headers: {
    id: 'ID', name: 'Full Name',
    email: 'Email', role: 'Role',
  },
  formatters: {
    salary: ['currency', 'USD'],
  },
  hasSelection: true,
  hasActions: true,
  actions: [
    { type: 'view', label: 'View', icon: 'Eye',
      action: (row) => console.log(row) },
  ],
  filters: [
    { field: 'role', type: 'select',
      options: [{ label: 'Admin', value: 'admin' }] },
  ],
  globalSearch: { enabled: true, mode: 'fuzzy' },
});
// Template: <hk-table [config]="table" [data]="users()" />

Provider Service Purpose

PAGINATION HELPERS

Server-Side Ready

Signal-based pagination state management for both cursor and offset strategies.

Cursor Cursor-Based Pagination 

const pagination = createCursorPagination({
  pageSize: 20,
});

// Computed signals
pagination.canGoNext   // Signal<boolean>
pagination.canGoPrev   // Signal<boolean>
pagination.isLoading   // Signal<boolean>

// Get HttpParams for your API
const params = getCursorHttpParams(pagination);

Offset Offset-Based Pagination 

const pagination = createOffsetPagination({
  pageSize: 25,
  totalItems: 500,
});

// Navigate
pagination.goNext()
pagination.goToPage(3)
pagination.changePageSize(50)

// Computed: totalPages, offset, etc.
pagination.totalPages  // Signal<number>

GET STARTED

Up and Running in Minutes

1

Configure the registry

Add this to your project's .npmrc file:

@hakistack:registry=https://hakistack-registry.fly.dev
2

Install the package

npm install @hakistack/ng-daisyui
3

Import and use

import { DynamicFormComponent, createForm, field }
  from '@hakistack/ng-daisyui';

@Component({
  imports: [DynamicFormComponent],
  template: `
    <hk-dynamic-form [config]="form.config()" />
    <button (click)="form.submit()">Submit</button>
  `,
})
export class MyComponent {
  form = createForm({
    fields: [field.text('name', 'Name')],
    onSubmit: (data) => console.log(data),
  });
}
